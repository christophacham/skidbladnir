    Ensure the search flow is fully wired end-to-end. Most of the infrastructure exists from Plan 01 (searchQuery in taskStore, matchingIds derived, dimmed prop on TaskCard). This task verifies and completes the wiring.

    **Verify NavBar search input:**
    - The search input in NavBar binds to `taskStore.searchQuery`
    - Pressing '/' (from global keyboard handler) focuses the input via `uiStore.searchFocused`
    - Pressing Escape in the search input: clears `taskStore.searchQuery` to '', blurs the input, sets `uiStore.searchFocused = false`
    - Placeholder changes to show current filter state (e.g., "Search tasks... (/)" when empty, shows result count when active: "{N} matches")

    **Verify Board -> Column -> TaskCard dimming chain:**
    - Board passes `searchActive={taskStore.searchQuery.trim().length > 0}` and `matchingIds={taskStore.matchingIds}` to each Column
    - Column passes `dimmed={searchActive && !matchingIds.has(task.id)}` to each TaskCard
    - TaskCard applies `class:opacity-30={dimmed}` with `transition-opacity duration-150`

    **Critical: Cards are DIMMED, not HIDDEN.** Per user decision, non-matching cards remain visible at 30% opacity to maintain spatial context. Do NOT use `{#if}` to filter cards out of the DOM. ALL cards always render. Only opacity changes.

    **Search matching logic (in taskStore):**
    - Case-insensitive substring match on `task.title` and `task.description`
    - Match if the query appears anywhere in either field
    - Empty query = all tasks match (no dimming)
    - This is simple text search, not fuzzy search. Fuzzy search (fuse.js) is used only in the command palette (Plan 03).

    **Edge cases:**
    - Tasks with null description: only match on title
    - Search while columns are collapsed: collapsed columns still show their task count badge (total count, not filtered count -- so the user knows tasks exist even if dimmed)
