    **commands.svelte.ts:**

    Create a command registry store. Define a `Command` interface:

    ```typescript
    interface Command {
      id: string;
      label: string;           // Display text
      shortcut?: string;       // Keyboard shortcut hint (e.g., "o", "e", "/")
      category: string;        // Group label (e.g., "Tasks", "Navigation", "View")
      action: () => void;      // What to execute
      keywords?: string[];     // Extra search terms
    }
    ```

    Create `CommandStore` class:
    - `commands = $state<Command[]>([])` -- the registry
    - `register()` method to populate commands
    - Call `register()` on construction

    Register these commands (per user decision -- "create task, switch project, search tasks, toggle sidebar, etc."):

    Tasks category:
    - "Create new task" (shortcut: "o") -> `uiStore.openCreateModal()`
    - "Search tasks" (shortcut: "/") -> `uiStore.focusSearch()`

    Navigation category:
    - "Toggle project sidebar" (shortcut: "e") -> `uiStore.toggleSidebar()`
    - One entry per project: "Switch to {project.name}" -> `projectStore.setActive(id)`. These are dynamic -- rebuild when `projectStore.list` changes using `$effect`.

    View category:
    - "Collapse all columns" -> collapse all in uiStore
    - "Expand all columns" -> expand all in uiStore

    Export singleton `commandStore`.

    **CommandPalette.svelte:**

    Render a modal overlay when `uiStore.commandPaletteOpen` is true.

    Layout:
    - Overlay: fixed position, full viewport, semi-transparent black background, z-index highest
    - Dialog: centered horizontally, positioned ~20% from top (not vertically centered -- feels more like VS Code), ~500px wide
    - Background: `--color-surface`, border: `--color-popup-border`, rounded, shadow

    Content:
    - Search input at top: autofocus, placeholder "Type a command...", styled with `--color-bg` background
    - Results list below: filtered commands using fuse.js for fuzzy matching
    - Each result: show `label` with optional `shortcut` badge right-aligned, and `category` as small dimmed text
    - Highlighted result: `--color-surface-hover` background
    - Max height with overflow scroll: show up to ~8-10 results

    Fuzzy search with fuse.js:
    ```typescript
    import Fuse from 'fuse.js';
    // Configure fuse with keys: ['label', 'keywords', 'category']
    // threshold: 0.4 (moderate fuzziness)
    ```
    When query is empty, show all commands. When query has text, show fuse results sorted by score.

    Keyboard navigation:
    - Arrow Up/Down: navigate highlighted result
    - Enter: execute highlighted command's `action()`, close palette
    - Escape: close palette without executing
    - Type to filter: update fuse search query

    On action execution:
    - Call `command.action()`
    - Call `uiStore.toggleCommandPalette()` to close
    - Clear the search query

    **Update +page.svelte:**
    - Import and render CommandPalette component (renders as overlay, position doesn't matter in DOM but place after Board and modals for readability)

    **Keyboard integration (already in +layout.svelte from Plan 01):**
    - Ctrl+K / Meta+K toggles command palette (already wired in Plan 01's keyboard handler)
    - Verify `event.preventDefault()` is called to prevent browser's default Ctrl+K behavior
