<objective>
Add structured multi-layer logging and config hot-reload to the agtxd daemon.

Purpose: The daemon needs production-grade observability (JSON log files for machine parsing, pretty stderr for development) and the ability to change configuration without restarts. These are operational necessities before the daemon can manage long-lived agent processes in Phase 2.

Output: Daemon writes rotating JSON logs to `~/.local/share/agtx/logs/`, displays pretty logs on stderr, watches config.toml for changes, and dynamically reloads log levels and safe config values.
</objective>

  <behavior>
    - DaemonConfig struct has port (u16, default 3742), bind (String, default "127.0.0.1"), log_level (String, default "info")
    - GlobalConfig gains a `daemon: DaemonConfig` field with `#[serde(default)]`
    - Existing config files without [daemon] section parse successfully (backwards compatible)
    - init_logging creates log files at the specified directory with daily rotation
    - Log files contain JSON-formatted structured entries (one JSON object per line)
    - stderr output is human-readable with ANSI colors
    - WorkerGuard is returned so caller can keep it alive
    - reload handle is returned so caller can change log level at runtime
  </behavior>
  <action>
    **Step 1: Add DaemonConfig to agtx-core config.**
    In `crates/agtx-core/src/config/mod.rs`, add:
    ```rust
    #[derive(Debug, Clone, Serialize, Deserialize)]
    pub struct DaemonConfig {
        #[serde(default = "default_port")]
        pub port: u16,
        #[serde(default = "default_bind")]
        pub bind: String,
        #[serde(default = "default_log_level")]
        pub log_level: String,
    }

    impl Default for DaemonConfig {
        fn default() -> Self {
            Self {
                port: default_port(),
                bind: default_bind(),
                log_level: default_log_level(),
            }
        }
    }

    fn default_port() -> u16 { 3742 }
    fn default_bind() -> String { "127.0.0.1".to_string() }
    fn default_log_level() -> String { "info".to_string() }
    ```
    Add `#[serde(default)] pub daemon: DaemonConfig` field to `GlobalConfig` and update its `Default` impl.

    **Step 2: Add tracing dependencies to agtxd/Cargo.toml.**
    Add `tracing-subscriber = { workspace = true }` and `tracing-appender = { workspace = true }`.

    **Step 3: Create crates/agtxd/src/logging.rs.**
    Implement `pub fn init_logging(log_dir: &Path, level: &str) -> Result<(ReloadHandle, WorkerGuard)>`:

    - Create `tracing_appender::rolling::daily(log_dir, "agtxd.log")` for daily rotation
    - Wrap in `tracing_appender::non_blocking()` to get `(non_blocking_writer, guard)`
    - Create a reloadable `EnvFilter` via `tracing_subscriber::reload::Layer::new(filter)`
    - Build subscriber with `tracing_subscriber::registry()`:
      - `.with(reload_filter_layer)` -- reloadable env filter
      - `.with(fmt::layer().json().with_writer(non_blocking_writer))` -- JSON to file
      - `.with(fmt::layer().pretty().with_writer(std::io::stderr))` -- pretty to stderr
    - Call `.init()` on the subscriber
    - Return the reload handle and worker guard
    - Define a type alias for the reload handle to keep the signature manageable

    Ensure the log directory is created with `std::fs::create_dir_all` before initializing the appender.

    The log directory path: use `directories::ProjectDirs` with `data_dir()` + `logs/` to get `~/.local/share/agtx/logs/` on Linux per user decision.

    **Step 4: Wire logging into main.rs.**
    At the very start of `main()`:
    ```rust
    let config = GlobalConfig::load().unwrap_or_default();
    let log_dir = GlobalConfig::data_dir()?.join("logs");
    let (_reload_handle, _guard) = logging::init_logging(&log_dir, &config.daemon.log_level)?;
    ```
    The `_guard` MUST be declared in main scope (not inside a block) so it lives for the entire program. The `_reload_handle` will be passed to the config watcher in Task 2.

    Update main.rs to read `config.daemon.port` and `config.daemon.bind` for the server bind address instead of hardcoded values.

    **Step 5: Write logging tests.**
    In `crates/agtxd/tests/logging_tests.rs`:
    - Test that init_logging creates the log directory if missing
    - Test that after init_logging, a `tracing::info!("test message")` call results in a JSON line in the log file (read the file, parse each line as JSON, find the test message)
    - Test that the returned reload handle can accept a new EnvFilter without panicking
  </action>

<task type="auto" tdd="true">
  <name>Task 2: Implement config file watcher with hot-reload</name>
  <files>
    crates/agtxd/Cargo.toml,
    crates/agtxd/src/config_watcher.rs,
    crates/agtxd/src/state.rs,
    crates/agtxd/src/main.rs,

  <behavior>
    - Config watcher detects changes to config.toml within 1 second
    - Log level changes are applied via the tracing reload handle
    - Safe config values (default_agent, theme) update in shared state
    - Structural values (port, bind) are NOT changed at runtime (logged as warning)
    - Debouncing prevents multiple reloads from a single save (editor write patterns)
    - Invalid config files are logged as warnings without crashing the daemon
    - Watcher continues monitoring after a parse error
  </behavior>
  <action>
    **Step 1: Add notify dependency.**
    In `crates/agtxd/Cargo.toml`, add `notify = { workspace = true }`.

    **Step 2: Update state.rs.**
    Add to `AppState`:
    ```rust
    pub config: Arc<RwLock<GlobalConfig>>,
    ```
    Use `tokio::sync::RwLock` (not std) since the lock may be held across await points. Import and store the shared config so handlers can read current config and the watcher can update it.

    **Step 3: Create crates/agtxd/src/config_watcher.rs.**
    Implement `pub async fn watch_config(config_path: PathBuf, shared_config: Arc<RwLock<GlobalConfig>>, reload_handle: ReloadHandle)`:

    - Create a `tokio::sync::mpsc::channel(1)` for notification
    - Create `notify::RecommendedWatcher` with a callback that sends to the channel on modify events
    - Watch the config file's parent directory (not the file itself -- editors may delete+recreate the file)
    - In the receive loop:
      - On event, sleep 200ms for debounce (drain any additional events during this window)
      - Read and parse the config file
      - On success:
        - Compare `new_config.daemon.log_level` with current; if changed, call `reload_handle.reload(EnvFilter::try_new(&new_level))`
        - Compare `new_config.daemon.port` and `.bind` with current; if changed, log `tracing::warn!("port/bind changes require restart")`
        - Update `*shared_config.write().await = new_config`
        - Log `tracing::info!("Configuration reloaded")`
      - On parse error: `tracing::warn!("Failed to reload config: {}", err)` and continue watching

    **Step 4: Wire watcher into main.rs.**
    After creating `AppState` and initializing logging:
    ```rust
    let config_path = GlobalConfig::config_path()?;
    let shared_config = state.config.clone();
    let watcher_reload_handle = reload_handle.clone();
    tokio::spawn(async move {
        config_watcher::watch_config(config_path, shared_config, watcher_reload_handle).await;
    });
    ```
    The reload handle type must be `Clone`-able. If the tracing reload handle is not Clone, wrap it in an `Arc`. Check the tracing-subscriber docs -- `reload::Handle` implements Clone when the inner layer implements Clone.

    **Step 5: Update API handlers to use live config.**
    Update the health endpoint to include `config.daemon.port` in its response (demonstrates live config reads). This is optional but useful for verifying config access works.

    **Step 6: Write config reload tests.**
    In `crates/agtxd/tests/config_reload_tests.rs`:
    - Test setup: Create a tempdir, write a valid config.toml, create shared state with Arc<RwLock<GlobalConfig>>
    - Test 1: Write a changed log_level to the config file, verify the shared config updates within 2 seconds
    - Test 2: Write an invalid TOML file, verify the shared config retains the old valid values (no crash, no corruption)
    - Test 3: Write a changed port value, verify it updates in the config but a warning was logged (check log output or just verify the value updates -- the warning is logged but hard to assert on in tests)

    For tests, use `tokio::time::timeout` to avoid hangs. The watcher task can be spawned in a tokio test runtime and cancelled after assertion.
  </action>
  <verify>
    <automated>cargo test -p agtxd --test config_reload_tests -- --nocapture 2>&1 | tail -20</automated>
  </verify>

Port/bind changes produce restart-required warnings. Debouncing prevents duplicate reloads. All config reload tests pass. Full workspace tests pass: `cargo test --workspace`.
