    **Step 2: Create error.rs.**
    Define `AppError` type that implements `IntoResponse` for axum. Use simple JSON format: `{"error": "message", "status": 404}`. Implement `From<anyhow::Error>` for `AppError` to map internal errors to 500. Create specific variants for NotFound (404) and BadRequest (400).

    **Step 3: Create state.rs.**
    Define `AppState` as a `#[derive(Clone)]` struct containing:
    - `db_path: PathBuf` (path to project database for task operations)
    - `global_db_path: PathBuf` (path to global index database)
    - `start_time: std::time::Instant` (for uptime calculation)
    Use `Database::open_project` path derivation pattern from existing code. For now, the daemon will use a default/configurable project path. Add a `Database::open_at(path: &Path)` constructor to `agtx-core/src/db/schema.rs` that opens a database at an explicit path (skip the hash-path derivation), for daemon use where the path comes from the global index.

    **Step 4: Create api/health.rs.**
    Handler: `async fn handler(State(state): State<AppState>) -> Json<Value>` returning `{"status": "healthy", "uptime_secs": N, "version": "0.1.0"}`. Use `env!("CARGO_PKG_VERSION")` for version.

    **Step 5: Create api/tasks.rs.**
    Create `pub fn router() -> Router<AppState>` with routes:
    - `GET /` -> `list_tasks`: Opens database via `spawn_blocking`, calls `db.get_all_tasks()`, returns JSON array
    - `POST /` -> `create_task`: Accepts `Json<CreateTaskRequest>` with `{title, agent, project_id, description?}`, creates `Task::new()`, saves via `db.create_task()`, returns 201 with task JSON
    - `GET /{id}` -> `get_task`: Returns task JSON or 404
    - `PUT /{id}` -> `update_task`: Accepts `Json<UpdateTaskRequest>`, updates fields, returns task JSON
    - `DELETE /{id}` -> `delete_task`: Deletes and returns 204, or 404

    All database calls wrapped in `tokio::task::spawn_blocking` since rusqlite is synchronous.

    **Step 6: Create api/projects.rs.**
    Create `pub fn router() -> Router<AppState>` with routes:
    - `GET /` -> `list_projects`: Opens global DB, returns project list as JSON array
    - `GET /{id}` -> `get_project`: Returns project JSON or 404

    **Step 7: Create api/mod.rs.**
    Compose the full router:
    ```rust
    pub fn api_router() -> Router<AppState> {
        Router::new()
            .route("/health", get(health::handler))
            .nest("/api/v1/tasks", tasks::router())
            .nest("/api/v1/projects", projects::router())
    }
    ```

    **Step 8: Create shutdown.rs.**
    Implement `pub async fn shutdown_signal()` using tokio::signal:
    - Listen for ctrl_c and SIGTERM (unix) using `tokio::select!`
    - Log the signal received via `tracing::info!`

    **Step 9: Create main.rs.**
    - Parse CLI args for optional `--port` and `--bind` overrides
    - Load `GlobalConfig` to get daemon settings (default port 3742, bind 127.0.0.1)
    - Construct `AppState`
    - Build router with `tower_http::trace::TraceLayer` middleware
    - Bind `TcpListener` and serve with `axum::serve(...).with_graceful_shutdown(shutdown_signal())`
    - Print startup message to stderr: "agtxd listening on {bind}:{port}"

    **Step 10: Write tests in crates/agtxd/tests/api_tests.rs.**

    - Project listing against a tempdir database
