<objective>
Build the daemon-side workflow engine that orchestrates all phase transitions with their side effects, plus REST endpoints for advance, diff, PR creation, and plugin listing.

Purpose: The daemon currently has raw session/task CRUD but no concept of "workflow." This plan introduces the WorkflowService layer that coordinates multi-step transitions (create worktree, deploy skills, spawn agent PTY, send commands) and exposes it through API endpoints.

Output: WorkflowService in `crates/agtxd/src/workflow/`, four new API endpoints in `crates/agtxd/src/api/workflow.rs`, session_id DB migration, artifact polling task.
</objective>

**3. Create transitions module:**

Create `crates/agtxd/src/workflow/transitions.rs` with these async functions:

`backlog_to_planning(task, project, session_manager, db_path) -> Result<()>`:
- Resolve plugin: `WorkflowPlugin::load(plugin_name, Some(project_path))` falling back to `skills::load_bundled_plugin(plugin_name)`, all in spawn_blocking
- Stamp plugin on task if not set (use project config default or "agtx")
- Create git worktree via spawn_blocking: `RealGitOps.create_worktree(project_path, &slug)`
- Initialize worktree via spawn_blocking: `RealGitOps.initialize_worktree(...)` with merged copy_files from plugin + project
- Deploy skills to worktree via spawn_blocking: port `write_skills_to_worktree()` logic (canonical .agtx/skills/ + agent-native paths for all 5 agent formats). Reference src/tui/app.rs lines 5895-5983 for exact logic.
- Run plugin init_script if configured (spawn_blocking with `sh -c`)
- Build agent command: `Agent::build_interactive_command("")` (no prompt, skill/prompt sent separately)
- Spawn PTY session: `session_manager.spawn(SpawnRequest { command: agent_shell, args, working_dir: worktree_path, rows: 24, cols: 80, env: {} })`
- Update task: `session_id = uuid.to_string()`, `worktree_path`, `branch_name = "task/{slug}"`
- Save task to DB
- Wait for agent readiness: subscribe to broadcast, wait 3s of output silence or 10s timeout
- Resolve skill command via `skills::transform_plugin_command()` for agent type
- Resolve prompt via template substitution ({task}, {task_id}, {phase})
- Send command + prompt via `session_manager.write()` (for gemini/codex: combine as single message with newline separator; for others: send command, wait 2s, send prompt)

`planning_to_running(task, project, session_manager, db_path) -> Result<()>`:
- Load plugin, resolve "running" skill command + prompt for the agent
- If task has session_id: write command/prompt via session_manager.write()
- If no session_id: spawn new session (same as backlog_to_planning but with "running" phase)

`running_to_review(task, db_path) -> Result<()>`:
- Just update status (per user decision: no forced PR prompt)
- Copy-back files if plugin.copy_back has "review" entries (spawn_blocking)

`review_to_done(task, project, db_path) -> Result<()>`:
- Check PR status if task.pr_number is set: spawn_blocking with `RealGitHubOps.get_pr_state()`
- If PR is open, include warning in result (not a blocker per user decision)
- Kill PTY session if still alive: `session_manager.kill(uuid)`
- Remove worktree via spawn_blocking (keep branch): `RealGitOps.remove_worktree()`
- Clear session_id on task

`review_to_planning_cyclic(task, project, session_manager, db_path) -> Result<()>`:
- Increment task.cycle
- Same as backlog_to_planning but reuse existing worktree (don't recreate)
- Send planning command for the new cycle

Helper `generate_task_slug(id: &str, title: &str) -> String` -- port from TUI.
Helper `resolve_prompt(plugin, phase, task_content, task_id, cycle) -> String` -- port from TUI (lines 5274-5322).
Helper `resolve_skill_command(plugin, phase, agent_name, task_content, cycle) -> Option<String>` -- port from TUI (lines 5326-5370).
Helper `write_skills_to_worktree(worktree_path, project_path, plugin, agent_names)` -- port from TUI (lines 5895-5983). This is a blocking function, call via spawn_blocking.

Create `crates/agtxd/src/api/workflow.rs` with four endpoints:

**1. POST /api/v1/workflow/tasks/{id}/advance**
```rust
#[derive(Deserialize)]
struct AdvanceRequest {
    #[serde(default = "default_next")]
    direction: String, // "next" or "cycle"
}
```
- Extract task_id from path, direction from JSON body
- Call `state.workflow.advance_task(&id, &req.direction).await`
- For Backlog->Planning: return 200 with updated task immediately. The heavy setup (worktree, agent spawn) runs in background via tokio::spawn inside WorkflowService. Frontend sees status change to Planning instantly; session_id populated later.
- Return JSON response with `{ task: Task, warning: Option<String> }` (warning for open PR on review->done)

**2. GET /api/v1/workflow/tasks/{id}/diff**
```rust
#[derive(Serialize)]
struct DiffResponse { diff: String }
```
- Load task from DB (spawn_blocking)
- Get worktree_path from task (400 if no worktree)
- Run `git diff main --unified=3` in worktree via spawn_blocking
- Also collect untracked file diffs: run `git ls-files --others --exclude-standard`, then for each untracked file `git diff --no-index /dev/null {file}`
- Concatenate all diffs
- Return `{ diff: raw_unified_diff_string }`

**3. POST /api/v1/workflow/tasks/{id}/pr**
```rust
#[derive(Deserialize)]
struct CreatePrRequest {
    title: String,
    body: String,
    base: Option<String>, // defaults to "main"
}
#[derive(Serialize)]
struct PrResponse { pr_number: i32, pr_url: String }
```
- Load task from DB, get worktree_path + branch_name
- Stage all changes, commit with task title, push branch (all via spawn_blocking with RealGitOps)
- Create PR via spawn_blocking with `RealGitHubOps.create_pr(project_path, title, body, branch)`
- Update task: pr_number, pr_url, save to DB
- Return `{ pr_number, pr_url }`

**4. GET /api/v1/workflow/tasks/{id}/pr/generate**
- Load task from DB, get worktree_path, agent
- Run `git diff main` to get diff (spawn_blocking)
- Use `AgentOperations::generate_text()` to generate PR title + body from diff (spawn_blocking). Use the task's configured agent. Prompt: "Generate a concise PR title and description for these changes:\n\n{diff}"
- Parse response: first line = title, rest = body
- Return `{ title: String, body: String }`
- If generate_text fails, return `{ title: "", body: "" }` (per user decision: empty for manual entry on failure)

**5. GET /api/v1/workflow/plugins**
```rust
#[derive(Serialize)]
struct PluginInfo { name: String, description: String, source: String }
```
- Collect bundled plugins from `skills::BUNDLED_PLUGINS` (name, description, "bundled")
- Scan global plugins dir `~/.config/agtx/plugins/*/plugin.toml` (spawn_blocking)
- Scan project-local plugins if project context available (spawn_blocking)
- Return Vec<PluginInfo> with source precedence noted

**6. GET /api/v1/workflow/tasks/{id}/pr/status**
- Load task, check pr_number exists
- Call `RealGitHubOps.get_pr_state()` via spawn_blocking
- Return `{ state: "open"|"merged"|"closed"|"unknown" }`
