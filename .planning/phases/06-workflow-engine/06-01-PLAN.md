<objective>
Build the daemon-side workflow engine that orchestrates all phase transitions with their side effects, plus REST endpoints for advance, diff, PR creation, and plugin listing.

Purpose: The daemon currently has raw session/task CRUD but no concept of "workflow." This plan introduces the WorkflowService layer that coordinates multi-step transitions (create worktree, deploy skills, spawn agent PTY, send commands) and exposes it through API endpoints.

Output: WorkflowService in `crates/agtxd/src/workflow/`, four new API endpoints in `crates/agtxd/src/api/workflow.rs`, session_id DB migration, artifact polling task.
</objective>

`review_to_planning_cyclic(task, project, session_manager, db_path) -> Result<()>`:

**1. POST /api/v1/workflow/tasks/{id}/advance**

```rust
struct AdvanceRequest {

```

- Call `state.workflow.advance_task(&id, &req.direction).await`
- For Backlog->Planning: return 200 with updated task immediately. The heavy setup (worktree, agent spawn) runs in background via tokio::spawn inside WorkflowService. Frontend sees status change to Planning instantly; session_id populated later.

**2. GET /api/v1/workflow/tasks/{id}/diff**
```rust
#[derive(Serialize)]
struct DiffResponse { diff: String }
```
- Load task from DB (spawn_blocking)
- Get worktree_path from task (400 if no worktree)
- Run `git diff main --unified=3` in worktree via spawn_blocking
- Also collect untracked file diffs: run `git ls-files --others --exclude-standard`, then for each untracked file `git diff --no-index /dev/null {file}`
- Concatenate all diffs
- Return `{ diff: raw_unified_diff_string }`

**3. POST /api/v1/workflow/tasks/{id}/pr**
```rust
#[derive(Deserialize)]
struct CreatePrRequest {

    body: String,
    base: Option<String>, // defaults to "main"
}
#[derive(Serialize)]
struct PrResponse { pr_number: i32, pr_url: String }
```
