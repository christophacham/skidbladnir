**2. Create `web/src/lib/components/DiffView.svelte`:**

A component that fetches the raw diff from the daemon and renders it with syntax highlighting.

Props: `taskId: string`

**Diff parsing logic (inline, no external library):**

Parse unified diff format into file sections:
```typescript
interface DiffFile {
    path: string;
    hunks: DiffHunk[];
}
interface DiffHunk {
    header: string;      // @@ -1,3 +1,5 @@
    lines: DiffLine[];
}
interface DiffLine {
    content: string;
    type: 'add' | 'remove' | 'context';
}
```

Parser: Split on `^diff --git` or `^--- a/` markers. For each file section, extract file path from `+++ b/path`. Split hunks on `^@@` lines. Classify each line by prefix: `+` = add, `-` = remove, ` ` = context.

**Language detection:**

Map file extensions to Shiki language names:
```typescript
const EXT_MAP: Record<string, string> = {
    rs: 'rust', ts: 'typescript', tsx: 'tsx', js: 'javascript', jsx: 'jsx',
    py: 'python', svelte: 'svelte', toml: 'toml', yaml: 'yaml', yml: 'yaml',
    json: 'json', md: 'markdown', css: 'css', html: 'html', sh: 'bash',
    go: 'go', rb: 'ruby', java: 'java', c: 'c', cpp: 'cpp', h: 'c',
};
function detectLang(path: string): string {
    const ext = path.split('.').pop() || '';
    return EXT_MAP[ext] || 'text';
}
```

**Shiki highlighting:**

Use `createHighlighter()` with explicit language list to control bundle size (per RESEARCH.md Pitfall 5). Create highlighter once (lazy singleton):
```typescript
import { createHighlighter, type Highlighter } from 'shiki';

let highlighterPromise: Promise<Highlighter> | null = null;
function getHighlighter(): Promise<Highlighter> {
    if (!highlighterPromise) {
        highlighterPromise = createHighlighter({
            themes: ['github-dark'],
            langs: ['rust', 'typescript', 'javascript', 'python', 'svelte', 'toml', 'yaml', 'json', 'markdown', 'css', 'html', 'bash', 'tsx', 'jsx']
        });
    }
    return highlighterPromise;
}
```

For each hunk in each file:
- Strip the +/- prefix from lines to get clean code for highlighting
- Highlight the entire hunk content as a single block using `highlighter.codeToHtml(code, { lang, theme: 'github-dark' })`
- The output is pre-highlighted HTML. Wrap each line in a div with class `diff-add`, `diff-remove`, or `diff-context` for background coloring.

Alternative simpler approach: render each file's changes as a single code block with line decorations:
- Collect all lines (stripped of +/- prefix) into one string
- Use Shiki decorations API to add classes per line:
```typescript
const decorations = lines.map((line, i) => ({
    start: { line: i, character: 0 },
    end: { line: i, character: line.content.length },
    properties: { class: `diff-${line.type}` }
}));
const html = highlighter.codeToHtml(code, { lang, theme: 'github-dark', decorations });
```

**CSS classes:**
```css
:global(.diff-add) { background-color: rgba(46, 160, 67, 0.15); }
:global(.diff-remove) { background-color: rgba(248, 81, 73, 0.15); }
:global(.diff-context) { opacity: 0.7; }
```

**Component structure:**
- On mount / when taskId changes: fetch diff via `fetchDiff(taskId)`
- Show loading spinner while fetching
- If diff is empty: show "No changes" message
- Parse diff into files, render each file with:
  - File path header (styled like GitHub diff headers)
  - Highlighted hunks with +/- line coloring
- Scroll container for long diffs

**Loading state:** Show a pulsing "Loading diff..." while fetching + highlighting (Shiki init can take a moment on first use).

**Error handling:** If fetchDiff fails (e.g. no worktree), show "Unable to load diff" with the error message.
