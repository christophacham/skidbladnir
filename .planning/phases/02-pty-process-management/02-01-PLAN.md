<objective>
Build the core session management module for PTY process lifecycle.

Purpose: This is the foundational layer that all PTY operations depend on. It provides the ability to spawn agent processes with PTY pairs, capture their output continuously, send input, resize terminals, and track PIDs. Without this, no REST endpoints or cleanup logic can function.

Output: A `session` module in `crates/agtxd/src/session/` with types, output persistence, and a fully functional SessionManager. Integration tests verify spawn/read/write/resize/PID tracking using real child processes (simple shell commands, not agents).
</objective>

  <action>
    Implement `SessionManager` in `crates/agtxd/src/session/manager.rs`:

    **Structure:**
    ```rust
    pub struct SessionManager {
        sessions: Arc<RwLock<HashMap<Uuid, SessionHandle>>>,
        sessions_dir: PathBuf,  // ~/.local/share/agtx/sessions/
    }
    ```

    **Constructor:**
    `SessionManager::new(sessions_dir: PathBuf) -> Self` -- Create the directory if it does not exist. Initialize empty sessions HashMap.

    **spawn(&self, req: SpawnRequest) -> anyhow::Result<Uuid>:**
    1. Generate UUID via `Uuid::new_v4()`
    2. Create output file path: `{sessions_dir}/{uuid}.log`
    3. Create `SessionOutput::new(&output_path)`
    4. Allocate PTY: `Pty::new()?`; get pts: `pty.pts()?`; resize: `pty.resize(Size::new(req.rows, req.cols))?`
    5. Build command: `pty_process::Command::new(&req.command)` with `.args(&req.args)`, `.current_dir(&req.working_dir)`, `.env("TERM", "xterm-256color")`. Add any extra envs from `req.env`.
    6. Configure PR_SET_PDEATHSIG in unsafe pre_exec: `libc::prctl(libc::PR_SET_PDEATHSIG, libc::SIGTERM)`. Return Err if prctl returns -1.
    7. Spawn: `cmd.spawn(&pts)?` -- this consumes pts, closing slave fd in parent.
    8. Get PID from `child.id().unwrap_or(0)` as u32.
    9. Split pty into read/write halves via `pty.into_split()`. This gives `(OwnedReadPty, OwnedWritePty)`.
    10. Wrap output in `Arc<RwLock<SessionOutput>>`.
    11. Spawn reader task via `tokio::spawn(reader_task(read_pty, output.clone(), session_id))`.
    12. Create `SessionHandle` with all fields, state = Running.
    13. Insert into sessions map.
    14. Log with tracing: `info!(session_id = %id, pid = pid, "Session spawned")`
    15. Return Ok(id).

    **reader_task (private async fn):**
    Read from `OwnedReadPty` in a loop with 4096-byte buffer. On Ok(0) (EOF): break. On Ok(n): `output.write().await.append(&buf[..n]).await`. On Err: log error, break. After loop exits, this task completes.

    **write(&self, id: Uuid, input: &str) -> anyhow::Result<()>:**
    Get session handle (read lock). Write `input.as_bytes()` then `b"\n"` then flush to `write_pty`. Use `tokio::io::AsyncWriteExt`. Return error if session not found.

    **resize(&self, id: Uuid, rows: u16, cols: u16) -> anyhow::Result<()>:**
    Get session handle. Call `handle.write_pty.resize(Size::new(rows, cols))?`. pty-process `OwnedWritePty` supports resize. If it doesn't, keep a reference to the original Pty. (Test during implementation -- RESEARCH.md open question #2.)

    **get(&self, id: Uuid) -> Option<SessionInfo>:**
    Read lock on sessions. Map handle to SessionInfo (id, pid, state, created_at, output.read().total_bytes()).

    **list(&self) -> Vec<SessionInfo>:**
    Read lock. Map all handles to SessionInfo.

    **get_output(&self, id: Uuid) -> Option<Vec<u8>>:**
    Read lock. Return output.read().tail().

    **kill(&self, id: Uuid) -> anyhow::Result<()>:**
    Write lock. Remove handle from map. Call `handle.child.kill().await?`. Call `handle.child.wait().await?` to reap. Abort reader task via `handle.reader_handle.abort()`. Log session killed.

    **shutdown_all(&self):**
    Write lock. Drain all sessions. For each: kill child, wait to reap, abort reader. This is called during daemon shutdown. Log each.

    **Add tests** for the 6 behaviors listed above. Tests spawn simple commands:
    - Test 1: Spawn `sh -c "sleep 10"` (long-running), verify list/get.
    - Test 2: Spawn `sh -c "echo hello && sleep 1"`, sleep 500ms, check output.
    - Test 3: Spawn `cat`, write input, sleep 500ms, check output.
    - Test 4: Spawn `sh -c "sleep 10"`, resize, verify Ok.
    - Test 5: Spawn `sh -c "echo $$"`, check PID is non-zero.
    - Test 6: Spawn `sh -c "sleep 10"`, kill, verify state is Exited.

    Use `tokio::time::sleep` for waits. Clean up all sessions in test teardown. Use `tempfile::tempdir()` for sessions_dir.

    IMPORTANT: The `write_pty` field in SessionHandle needs interior mutability for write operations. Since `SessionHandle` is behind `RwLock<HashMap<...>>`, writes need a write lock on the map. Alternatively, wrap `OwnedWritePty` in `tokio::sync::Mutex` for finer-grained locking. Use `Mutex<OwnedWritePty>` in SessionHandle so write() only needs a read lock on the sessions map.

    IMPORTANT: For resize, `OwnedWritePty` in pty-process 0.5 does implement `resize()`. If it doesn't compile, fall back to storing the `Pty` behind a Mutex instead of splitting.
