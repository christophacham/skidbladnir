<objective>
Wire session management into the daemon's REST API and shutdown lifecycle.

Purpose: The session module from Plan 01 provides internal PTY management but is not accessible via HTTP and is not integrated into the daemon's startup/shutdown flow. This plan exposes all session operations as REST endpoints and ensures all sessions are cleaned up when the daemon exits -- the critical safety requirement (PTY-05).

Output: REST API routes under `/api/v1/sessions/`, SessionManager in AppState, graceful shutdown integration, and HTTP-level integration tests.
</objective>

    **Step 1: Refactor shutdown to accept SessionManager.**

    The current `shutdown.rs` only provides `shutdown_signal()` which is used with `axum::serve().with_graceful_shutdown()`. The graceful shutdown signal fires, axum stops accepting connections, but there's no hook to kill sessions.

    Modify `crates/agtxd/src/main.rs` to:
    1. Clone the `Arc<SessionManager>` before passing state to the router.
    2. After `axum::serve(...).with_graceful_shutdown(shutdown::shutdown_signal()).await?`, call `session_manager.shutdown_all().await` BEFORE the `tracing::info!("agtxd shut down cleanly")` line.

    This ensures: signal received -> axum stops accepting -> existing requests drain -> then sessions killed -> then exit.

    The sequence in main becomes:
    ```rust
    let session_manager_for_shutdown = session_manager.clone();

    // ... build app, serve ...

    axum::serve(listener, app)
        .with_graceful_shutdown(shutdown::shutdown_signal())
        .await
        .context("Server error")?;

    // Clean up all active sessions after server stops
    tracing::info!("Shutting down active sessions...");
    session_manager_for_shutdown.shutdown_all().await;
    tracing::info!("agtxd shut down cleanly");
    ```

    **Step 2: Verify PR_SET_PDEATHSIG is already configured.**

    The `spawn()` method in manager.rs (from Plan 01) already configures `pre_exec` with `PR_SET_PDEATHSIG(SIGTERM)`. This is the kernel-level safety net. No additional work needed here -- just verify it's present in the Plan 01 implementation. If Plan 01 executor omitted it, add it now.

    **Step 3: Add shutdown cleanup test.**

    In `crates/agtxd/tests/session_tests.rs`, add a test:
    - `test_shutdown_all_kills_all_sessions`: Create a SessionManager, spawn 3 sessions (each running `sleep 30`), call `shutdown_all()`, verify that `list()` returns empty and no zombie processes remain.

    To verify no zombies: after shutdown_all, use `nix::sys::wait::waitpid(Pid::from_raw(pid), Some(WaitPidFlag::WNOHANG))` for each PID -- should return `Err(Errno::ECHILD)` (no such child, meaning it was reaped) or `Ok(WaitStatus::Exited(...))`.

    **Step 4: Add Drop implementation for SessionManager (defense in depth).**

    While `shutdown_all()` is the primary cleanup path, implement a synchronous best-effort cleanup in a `Drop` impl as a safety net. Since `Drop` cannot be async, use `std::process::Command::new("kill").arg("-TERM").arg(pid.to_string())` for each session that still has a PID. This is a last resort -- the explicit shutdown_all() is the correct path.

    Note: The Drop impl is fire-and-forget. It logs via tracing if sessions remain but does not block or panic.
