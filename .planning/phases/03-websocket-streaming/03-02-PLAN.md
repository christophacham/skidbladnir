**Extend crates/agtxd/src/api/ws.rs** (which already has ServerMessage/ClientMessage from Plan 01) with the WebSocket handler:

1. Add the `ws_handler` function -- the axum handler that validates the session and upgrades to WebSocket:

```rust
use axum::extract::ws::{Message, WebSocket, WebSocketUpgrade};
use axum::extract::{Path, Query, State};
use axum::response::IntoResponse;
use axum::http::StatusCode;
use futures_util::{SinkExt, StreamExt};
use base64::Engine;
use uuid::Uuid;

use crate::error::AppError;
use crate::session::{OutputEvent, SessionOutput, SessionState};
use crate::state::AppState;

#[derive(Debug, serde::Deserialize)]
pub struct WsQuery {
    /// Byte offset cursor for reconnection. If provided, server sends delta from this offset.
    #[serde(default)]
    pub cursor: Option<u64>,
}

pub async fn ws_handler(
    ws: WebSocketUpgrade,
    Path(id): Path<String>,
    Query(query): Query<WsQuery>,
    State(state): State<AppState>,
) -> Result<impl IntoResponse, AppError> {
    let uuid = Uuid::parse_str(&id)
        .map_err(|_| AppError::BadRequest(format!("Invalid UUID: {}", id)))?;

    // Validate session exists BEFORE upgrading
    let sub = state.session_manager.subscribe(uuid).await
        .ok_or_else(|| AppError::NotFound(format!("Session not found: {}", id)))?;

    Ok(ws.on_upgrade(move |socket| handle_ws(socket, uuid, state, sub, query.cursor)))
}
```

2. Implement `handle_ws` -- the core WebSocket connection handler:

   **Critical ordering per RESEARCH.md Pitfall 4:** Subscribe to broadcast BEFORE reading initial state to avoid missing output in the gap.

   - The `sub` tuple is `(broadcast::Receiver<OutputEvent>, Arc<RwLock<SessionOutput>>, PathBuf)` from `subscribe()`.
   - Split the WebSocket into sender/receiver halves using `futures_util::StreamExt::split()`.

   **Initial state delivery:**
   - If cursor is `Some(offset)` where offset > 0: read delta from file using `SessionOutput::read_range(output_path, offset, 1_048_576)` (1MB max). Send as one or more `ServerMessage::Output` messages.
   - If cursor is `None` or `Some(0)`: read ring buffer via `output.read().await.tail()`. Send as `ServerMessage::Output`.
   - After initial data, send `ServerMessage::Connected { session_id, total_bytes }` so client knows its new cursor position.

   **Send task (spawned):** Loop on `broadcast_rx.recv()`:
   - On `OutputEvent::Data { bytes, offset }`: base64-encode bytes, send `ServerMessage::Output { data, offset }` as `Message::Text`.
   - On `OutputEvent::StateChange(state)`: send `ServerMessage::State { session_state, exit_code }`.
   - On `RecvError::Lagged(n)`: log warning, send `ServerMessage::Error { message: "Missed {n} messages, may need to re-sync" }`, continue receiving.
   - On `sender.send()` error: break (client disconnected).

   **Receive task (spawned):** Loop on `receiver.next()`:
   - On `Message::Text(text)`: deserialize as `ClientMessage`.
     - `Write { input }`: call `state.session_manager.write_raw(session_id, input.as_bytes())`.
     - `Resize { rows, cols }`: call `state.session_manager.resize(session_id, rows, cols)`.
   - On `Message::Close(_)`: break.
   - On `Message::Ping(_)`: axum handles pong automatically, no action needed.
   - On parse error: send `ServerMessage::Error` with the parse error message.

   **Cleanup:** Use `tokio::select!` on both tasks. When either finishes, abort the other. No session state to clean up (broadcast receiver is dropped automatically).
